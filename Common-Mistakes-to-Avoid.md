# TCS NQT — Common Mistakes to Avoid

## ❌ Aptitude Mistakes

### Percentages
- ❌ Assuming x% increase then x% decrease = 0% change
- ✅ It's always a LOSS: Net = -x²/100%

### Time & Work
- ❌ Adding days instead of rates: 12 + 18 = 30 days
- ✅ Add RATES: 1/12 + 1/18 = 5/36, so 7.2 days

### Time, Speed & Distance
- ❌ Forgetting to convert km/hr to m/s
- ✅ km/hr × 5/18 = m/s

### Averages
- ❌ Using average for individual values
- ✅ Average is a summary, not actual values

### Profit & Loss
- ❌ Calculating profit on SP instead of CP
- ✅ Always calculate on Cost Price

### Ratios
- ❌ Mixing up which term is larger
- ✅ If a:b = 2:3, a is smaller

---

## ❌ Verbal Mistakes

### Sentence Completion
- ❌ Choosing based on "sounding right" without context
- ✅ Read the FULL sentence, check for contrast/cause words

### Error Identification
- ❌ Changing words that are already correct
- ✅ Check subject-verb agreement first

### Reading Comprehension
- ❌ Using outside knowledge to answer
- ✅ Answer ONLY from the passage

---

## ❌ Reasoning Mistakes

### Series
- ❌ Only checking differences (check ratios too!)
- ✅ Try multiple patterns: add, multiply, square

### Blood Relations
- ❌ Starting from the beginning of the statement
- ✅ Start from the END and work backward

### Seating Arrangement
- ❌ Not fixing one person's position first
- ✅ Anchor one person, then place others relative to them

### Coding-Decoding
- ❌ Assuming pattern is always +1
- ✅ Check all given examples to find the pattern

---

## ❌ Coding Mistakes

### Input/Output
- ❌ Not reading input correctly
- ✅ Use `input()`, `split()`, `map(int, ...)` properly

### Edge Cases
- ❌ Not handling empty input
- ✅ Always check: empty list, single element, negative numbers

### Time Limit Exceeded
- ❌ Using O(n²) when O(n) is possible
- ✅ Use HashMap for lookups, avoid nested loops

### Wrong Output Format
- ❌ Printing extra spaces or newlines
- ✅ Match exact output format specified

### Off-by-One Errors
- ❌ `range(n)` vs `range(1, n+1)`
- ✅ Double-check loop boundaries

### Not Testing
- ❌ Submitting without testing
- ✅ Run with given examples FIRST

---

## ❌ Exam Strategy Mistakes

### Time Management
- ❌ Spending 5+ minutes on one question
- ✅ If stuck > 1.5 min, flag and move on

### Not Attempting All
- ❌ Leaving questions blank
- ✅ NO negative marking — attempt EVERYTHING

### Reading Too Fast
- ❌ Missing key words like "NOT", "EXCEPT", "ALWAYS"
- ✅ Read questions carefully, underline keywords

### Panic on Hard Questions
- ❌ Getting stuck and wasting time
- ✅ Skip hard ones, come back later

### Not Using Options
- ❌ Solving from scratch every time
- ✅ Plug in answer choices to verify

### Last-Minute Changes
- ❌ Changing answers at the last second
- ✅ First instinct is usually right

## ✅ Pre-Exam Checklist

### Night Before
- [ ] Get 7-8 hours sleep
- [ ] Review cheat sheet (15 min max)
- [ ] Prepare ID and exam login
- [ ] Set alarm for exam day

### Morning of Exam
- [ ] Eat a good breakfast
- [ ] Drink water
- [ ] Arrive/log in early
- [ ] Deep breaths, stay calm

### During Exam
- [ ] Read all questions first (2 min)
- [ ] Start with easy ones
- [ ] Flag hard questions
- [ ] Attempt ALL questions
- [ ] Save coding work frequently
