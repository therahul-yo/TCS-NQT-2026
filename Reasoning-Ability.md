# Reasoning Ability — Shortcuts, Patterns & Strategies

## 🔢 Number Series

### Common Patterns
1. **Addition/Subtraction**: +2, +3, -5, etc.
2. **Multiplication/Division**: ×2, ×3, ÷2, etc.
3. **Square/Cube**: n², n³, n²+1, n²-1
4. **Fibonacci-like**: Each term = sum of previous two
5. **Alternating operations**: +2, ×2, +2, ×2...
6. **Prime numbers**: 2, 3, 5, 7, 11, 13...
7. **Difference of differences**: Look at 2nd level differences

### Shortcut Steps
1. Check differences between consecutive terms
2. If differences aren't constant, check ratios
3. Check if terms are squares/cubes ± something
4. Look for alternating patterns (odd/even positions)

### Example
**2, 6, 12, 20, 30, ?**
- Differences: 4, 6, 8, 10, ? → +2 each time
- Next difference = 12, Answer = 30 + 12 = **42**

## 🔤 Letter Series

### Alphabet Positions (MEMORIZE!)
```
A=1  B=2  C=3  D=4  E=5  F=6  G=7  H=8  I=9  J=10
K=11 L=12 M=13 N=14 O=15 P=16 Q=17 R=18 S=19 T=20
U=21 V=22 W=23 X=24 Y=25 Z=26
```

### EJOTY Trick (for quick reference)
- **E=5, J=10, O=15, T=20, Y=25**

### Common Patterns
1. **Position shift**: +1, +2, -1, etc.
2. **Reverse alphabet**: Z=1, Y=2, X=3...
3. **Skip patterns**: Every 2nd/3rd letter
4. **Pair patterns**: AB, CD, EF or AC, EG, IK

### Shortcut: Reverse Position
- Reverse position = 27 - Forward position
- A(1) → Z(26) = 27-1, B(2) → Y(25) = 27-2

## 🔐 Coding-Decoding

### Types

1. **Letter Shifting (Caesar Cipher)**
   - Each letter shifted by fixed number
   - CAT(+3) → FDW

2. **Number Coding**
   - A=1, B=2... or reverse
   - FIND patterns in given examples

3. **Substitution Coding**
   - "Apple" means "Chair", "Chair" means "Table"
   - "What do you sit on?" → Table (not Chair!)

4. **Symbol Coding**
   - Using @, #, $, % for letters

### Strategy
1. Look at the given code
2. Compare letter by letter with original
3. Find the pattern/rule
4. Apply the same rule to the new word

## 👨‍👩‍👧‍👦 Blood Relations

### Key Relations
```
Grandfather → Father → Son/Grandson
Grandmother → Mother → Daughter/Granddaughter
Uncle/Aunt → Cousin
Brother/Sister → Sibling
Father's brother = Uncle
Mother's sister = Aunt
Father's sister = Aunt
Mother's brother = Uncle
Brother's son = Nephew
Brother's daughter = Niece
Sister's son = Nephew
Sister's daughter = Niece
```

### Strategy: Draw a Family Tree
1. Start from the person mentioned last
2. Work backwards
3. Use symbols: □ = Male, ○ = Female, — = married, | = child
4. Use horizontal lines for same generation

### Shortcut
- "Pointing to X, he said..." → X is the reference point
- Work from the END of the statement backward

## 🪑 Seating Arrangements

### Types
1. **Linear**: People in a row
2. **Circular**: People around a table
3. **Facing center**: Left = anti-clockwise, Right = clockwise
4. **Facing outside**: Left = clockwise, Right = anti-clockwise

### Strategy
1. Fix ONE person's position first
2. Use definite clues first, then relative clues
3. Draw positions and fill gradually
4. Check all constraints before finalizing

### Circular Seating Tips
- **Facing center**: Person on your left is counterclockwise
- **Facing outside**: Person on your left is clockwise
- 8 people in circle: 4 opposite pairs

## 🧭 Distance & Directions

### Key Rules
- **N→E→S→W** (clockwise)
- **N→W→S→E** (counterclockwise)
- Right turn = 90° clockwise
- Left turn = 90° counterclockwise

### Shortcut: Pythagorean Theorem
For final distance when path has turns:
- Distance = √(net_x² + net_y²)

### Example
- 3 km North, 4 km East → Distance = √(3²+4²) = 5 km

### Shadow Trick
- **Morning**: Shadow falls WEST
- **Evening**: Shadow falls EAST
- **Noon**: Shadow falls SOUTH (in Northern Hemisphere)

## ❓ Syllogisms

### Types of Statements
- **All A are B** → Some A are B ✓, Some B are A ✓
- **No A are B** → Some A are not B ✓
- **Some A are B** → Some B are A ✓
- **Some A are not B** → Cannot conclude anything about B

### Venn Diagram Method
1. Draw circles for each set
2. Place them according to statements
3. Check which conclusions are always true

### Rules
- "All A are B" + "All B are C" → "All A are C"
- "Some A are B" + "All B are C" → "Some A are C"
- "All A are B" + "No B are C" → "No A are C"
- "Some A are B" + "No B are C" → "Some A are not C"

## 📊 Data Sufficiency

### Strategy
1. **Use Statement 1 alone** — is it enough?
2. **Use Statement 2 alone** — is it enough?
3. **Use both together** — is it enough?

### Answer Choices
- A: Statement 1 alone is sufficient
- B: Statement 2 alone is sufficient
- C: Both together are sufficient
- D: Each alone is sufficient
- E: Neither is sufficient

### Tips
- Don't actually solve — just check if you COULD solve
- Look for contradictions between statements
- If a statement gives you the answer directly, it's sufficient

## 🔍 Odd One Out

### Common Patterns
1. **Category**: 3 are fruits, 1 is vegetable
2. **Property**: 3 are even, 1 is odd
3. **Relation**: 3 have same relationship, 1 doesn't
4. **Function**: 3 serve same purpose, 1 different

### Strategy
- Find the common link among 3 options
- The one without this link is the answer

## 🔣 Symbols & Notations

### Strategy
1. **Replace symbols** with mathematical operations
2. **Follow BODMAS** strictly
3. **Check each option** one by one

### Example
If @ means +, # means -, $ means ×, % means ÷:
- 5 @ 3 # 2 $ 4 = 5 + 3 - 2 × 4 = 5 + 3 - 8 = 0

1. **Draw diagrams** whenever possible
2. **Start with definite information** first
3. **Eliminate impossible options** quickly
4. **Practice pattern recognition** daily
5. **Time management** — don't get stuck on one puzzle
6. **Attempt all** — no negative marking
